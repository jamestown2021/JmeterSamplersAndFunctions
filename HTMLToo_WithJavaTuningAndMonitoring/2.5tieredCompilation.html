JVM Compilers

Normally JVM uses tiered compilation in order to deliver the optimal performance. However we can make the JVM run in interpreter by turning off the"TieredCompilation" flag. This will make the JVM run the code only in the interpreted mode.
We can also make the JVM make the method complied/store the method in the code cache by setting the "CompileThreshold" value. By default the value is 10k and after the method has executed for 10k times the method will be native complied and plced in the code cache for execution.
